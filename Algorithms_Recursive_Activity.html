Recursive Algorithms Activity

Maze Path Finder: Route Optimization


Maze solving is a common activity in computer science. In fact, there are many serious robotics competitions that use it, trying to get the fastest time. In this activity, we will use a recursive algorithm to find the shortest path through a maze.

Example Maze


To make this activity simpler, no maze will have any loops. Because of this, we can derive the optimal path from any valid solution. To find a solution, we used the "follow the left wall" maze strategy. That is, it will move through the maze, and at every fork, it will always choose to turn left if it can. If it can't, then it will try to go straight, then right if it fails, then finally backwards if it is the only option. Because there are no loops, it will always find the finish eventually. As it moves through the maze, it will keep track of all decisions so that we can reduce it later.

Mapped Maze


As we are testing your ability to implement a recursive algorithm, and not your knowledge of maze solving, we give you all possible optimizations for three sequential movements:
  • Left -> Back -> Right = Back
  • Left -> Back -> Straight = Right
  • Right -> Back -> Left = Back
  • Straight -> Back -> Left = Right
  • Straight -> Back -> Straight = Back
  • Left -> Back -> Left = Straight

optiMAZE Block


What you will be doing in this activity is taking the long path that was mapped, and create a recursive algorithm to reduce it to the optimal path. You will do this by completing the optiMAZE block in the Mouse Sprite. We provided 4 different difficulty levels:
  • Easy: Already optimizes the path once. You need to make it repeatedly optimize it until it stops improving
  • Medium: Tests the first three moves for a possible reduction (see list ablove). You need to test every sequence of three moves in the path for possible reduction then repeat the entire thing until it no longer shortens the path
  • Hard: Tests the first three moves for a possible reduction of Left -> Back -> Right = Back. You need to test for every possible reduction (see list ablove) for every sequence of three moves in the path, then repeat the entire thing until it no longer shortens the path
  • Insane: optiMAZE is empty. You will need to implement the entire block yourself.
